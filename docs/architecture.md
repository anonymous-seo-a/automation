# 母艦システム アーキテクチャ設計書

## 1. 全体構成

```
[LINE] ←→ [nginx (HTTPS)] ←→ [Express :3000]
                                    ↓
                          [Task Interpreter]
                           (Sonnet 4.6 API)
                                    ↓
                          [Task Queue (SQLite)]
                                    ↓
                          [Agent Router]
                                    ↓
                    ┌───────────────────────────┐
                    │  soico最適化AI (Phase 1)   │
                    │  収益スカウトAI (Phase 2)   │
                    │  SNSエージェント (Phase 3)  │
                    │  メンタル安全装置 (Phase 4) │
                    └───────────────────────────┘
                                    ↓
                          [Executor]
                           ├→ コード生成 (Claude API)
                           ├→ サンドボックス実行
                           ├→ テスト
                           ├→ エラー自己修正 (最大5回)
                           └→ LINE報告
```

## 2. モデルルーティング

| 処理 | モデル | コスト/1M tokens |
|------|--------|-----------------|
| 指示解釈・タスク分解 | Sonnet 4.6 | $3 / $15 |
| コード生成・修正 | Sonnet 4.6 | $3 / $15 |
| SEO戦略判断 | Opus 4.6 | $5 / $25 |
| エラー分析(3回失敗後) | Opus 4.6 | $5 / $25 |
| ナレッジ更新判断 | Opus 4.6 | $5 / $25 |

## 3. リトライ戦略

| 回数 | モデル | 戦略 |
|------|--------|------|
| 1回目 | Sonnet | そのまま再実行 |
| 2回目 | Sonnet | エラー情報付加して再生成 |
| 3回目 | Opus | エラーパターン分析→アプローチ変更 |
| 4回目 | Opus | 問題分割して部分実行 |
| 5回目 | Opus | 最終試行。失敗→LINE報告 |

## 4. セキュリティ

| 項目 | 対策 |
|------|------|
| LINE認証 | Webhook署名検証 + User ID許可リスト |
| API Key | `.env` + `.gitignore` |
| VPSアクセス | SSH鍵認証のみ |
| ファイアウォール | ufw: 22, 80, 443のみ |
| サンドボックス | `/tmp`隔離 + 30秒タイムアウト |
| HTTPS | Let's Encrypt (certbot) |
| 予算管理 | 日次/月次上限チェック。超過で全停止 |

| 4 | 予算管理・監視 | 1日 |
| 5+ | 追加エージェント | 後日 |
