## 2026-03-29: 初期設計・実装

### 判断ログ

| 判断 | 選択 | 理由 | 代替案 |
|------|------|------|--------|
| DB | SQLite (better-sqlite3) | VPS 1台完結。規模的に十分 | PostgreSQL（過剰） |
| API呼び出し | fetch直接 | SDK依存を避ける。デバッグ容易 | @anthropic-ai/sdk |
| Webフレームワーク | Express 4 | LINE SDK互換。Claude Codeの精度高 | Fastify（LINE SDK非対応） |
| プロセス管理 | PM2 | ログ管理が楽。再起動が簡単 | systemd（設定が煩雑） |
| モデル | Sonnet 4.6主軸 + Opus 4.6判断用 | 月予算3万以内。精度十分 | Opus一本（予算超過） |
| VPS | ConoHa 4GB | Node.js常駐+エージェント複数に必要。テンプレート豊富 | さくらVPS（コスパ僅差） |

## ファイル変更時の影響範囲メモ

変更頻度が高いファイルと、変更時に注意すべき依存先。

### config.ts を変更した場合
→ 全モジュールが依存。新しい環境変数を追加したら `.env.example` も更新。

### claude/client.ts を変更した場合
→ interpreter, executor, soicoAgent が依存。
  レスポンス形式を変えたら全ての呼び出し元を確認。

### agents/soico/prompts.ts を変更した場合
→ soicoAgent.ts のみ依存。プロンプト改善は最も頻繁に行う作業。
  変更内容と効果をこのファイルに記録すること。

### queue/taskQueue.ts を変更した場合
→ webhook.ts と executor.ts が依存。
  TaskRow の型を変えたら両方を確認。

### db/migrations.ts を変更した場合
→ 既存のDBがある場合、マイグレーションが壊れる可能性。
  カラム追加は `ALTER TABLE` で別途対応するか、DBを再作成。
